# Session Handoffs

Documentation for session handoffs between agents or developers.

## Purpose

Handoff documents capture:
- Work completed in a session
- Pending tasks
- Important context for the next session
- Blockers or issues encountered

## Naming Convention

```
YYYY-MM-DD-topic.md
```

Example: `2026-01-07-ray-cluster-upgrade.md`

## Handoff Template

```markdown
# Handoff: [Topic]

**Date:** YYYY-MM-DD
**Author:** [Name/Agent]

## Completed
- [ ] Task 1
- [ ] Task 2

## Pending
- [ ] Task 3
- [ ] Task 4

## Context
Important information for the next session.

## Blockers
Any issues that need resolution.

## Files Modified
- path/to/file1.md
- path/to/file2.md
```
