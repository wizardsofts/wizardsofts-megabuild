# Documentation Archive

Historical documentation, completed migrations, audits, and status reports.

## Contents

| Directory | Description |
|-----------|-------------|
| [migrations/](migrations/) | Completed migration logs (phases, database) |
| [retrospectives/](retrospectives/) | Post-mortems and lessons learned |
| [audits/](audits/) | Security audits and reviews |
| [status-reports/](status-reports/) | One-time completion reports |

## Archive Policy

Documents are archived when:
1. Migration/project is complete
2. Status report is no longer active
3. Document is superseded by newer version
4. Audit is older than 6 months

## What Goes Where

| Document Type | Destination |
|---------------|-------------|
| `*_COMPLETE.md` | [status-reports/](status-reports/) |
| `*_READY.md` | [status-reports/](status-reports/) |
| `PHASE*` logs | [migrations/](migrations/) |
| Security audits | [audits/](audits/) |
| Post-mortems | [retrospectives/](retrospectives/) |

## Retention

- Archives are kept indefinitely for audit trail
- Consider moving to separate repo after 1 year

## Note

Archived documents are still searchable and linkable.
They are moved here to reduce clutter in active documentation.
